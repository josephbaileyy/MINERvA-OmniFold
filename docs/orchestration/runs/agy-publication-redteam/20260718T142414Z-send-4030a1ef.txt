### Enforced Protections & Dependency Rules
1. **One-Writer / No-Duplicate-Outputs:** The `interactive-first` policy forbids submitting an `sbatch` equivalent to a job that can fit an already-running interactive allocation. This prevents silent Lustre output-file corruption.
2. **Commit Gate:** No scientific compute arrays (Agent C endpoints, Agent E 12-playlist, Agent B PET UQ) can be submitted using explicit scheduler dependencies (`--dependency=afterok:...`) right now because the prerequisite steps include **human/verifier code-review gates**, not just SLURM job completions. A batch array is only safe to queue early when its direct prerequisite is purely computational and immutable.
3. **Owner Continuity:** Roles remain strictly bound to their UUIDs and namespaces. Agent E owns the unassigned C++ (Claude-personal), Agent A handles standard paths, Agent C handles FPS, and Agent B owns the PET Python interface.
