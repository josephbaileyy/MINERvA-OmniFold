Agent C's repair successfully implements the file-level census reads, identity/schema tests, and atomic swap logic. However, it fails on strict historical provenance binding, completeness enforcement, and collision safety.

### Exact Defects

1.  **Completeness Bypass (`--hash-files`)**: `inventory_complete()` does not enforce `--hash-files`. An invocation without it exits 0 with `complete=true`, violating the requirement that the complete manifest binds every file by SHA-256.
2.  **No Extra-File Detection**: The inventory loop checks only the expected 120 paths. It ignores any unexpected/stale `.root` files in `active_universe_5d/fps/`. The tests do not mock extras.
3.  **Aggregate Log Corroboration**: `corroborate_md5_in_logs` accepts the array if *any* logs contain the MD5. It fails to bind each of the 120 files to its *exact* producer log and log SHA-256. (For example, it would miss that task 0 was produced by `55961845` because it searches only the provided array job).
4.  **Observation-Time Launcher Misrepresentation**: `launcher_provenance` records `"content_sha256": sha256_file(launcher_abs)` even when `committed_at_submit` is False. This misrepresents today's uncommitted file content as the exact production-time content.
5.  **Non-Terminal `sacct` Acceptance**: `build_historical_provenance` parses the `sacct` State/ExitCode but does not fail-closed if the state is non-terminal (e.g., `FAILED`, `RUNNING`) or nonzero.
6.  **Collision-Unsafe Atomic Write**: `atomic_write_json` uses `tmp = path + ".tmp"`. If two concurrent instances run, one will clobber the other's temp file before the `os.replace`.

import tempfile
def atomic_write_json(path, obj):
    os.makedirs(os.path.dirname(path) or ".", exist_ok=True)
    fd, tmp = tempfile.mkstemp(dir=os.path.dirname(path), suffix=".json.tmp")
    with os.fdopen(fd, "w") as fh:
        json.dump(obj, fh, indent=2)
    os.replace(tmp, path)
