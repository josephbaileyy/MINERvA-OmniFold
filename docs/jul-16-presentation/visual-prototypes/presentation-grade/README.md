# Presentation-grade teaching sequences

Open `index.html` in a browser. No server or external assets are required,
except that the last marginalization build optionally loads the cleared real
MINERvA figure from the adjacent Claude design package.

These are expanded replacements for concepts 1–4 in the first prototype
gallery. Each is a short build sequence rather than a single metaphor.

## Sequences

1. Why response matrices stop scaling
2. The response matrix before it was histogrammed
3. A classifier is a density-ratio meter
4. One complete OmniFold iteration
5. What “bin at the end” actually buys
6. Marginalization as a non-negotiable validation

The on-screen slide is audience-facing. The panel below it contains the
speaker move and the reason for each build; those notes are not intended to
appear in the final deck.

## Scientific scope

- All scatter plots and matrix populations are generic toy illustrations.
- The `10 bins per axis` scaling example is arithmetic, not a claim about the
  production MINERvA binning.
- The classifier sequence uses the equal-training-prior relation
  `w(x) = p(data|x)/(1-p(data|x))` as the visual intuition.
- The algorithm sequence includes the second truth-space classifier and the
  native treatment of truth-only misses; it does not imply that weights are
  merely copied once and the algorithm stops.
- “Bin at the end” is phrased as many binnings and projections of stored or
  derived truth observables, not literally any arbitrary observable without a
  validation obligation.
