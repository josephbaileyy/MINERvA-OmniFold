Today I want to do three things: explain OmniFold from first principles, show exactly how I use it with the MINERvA open-data product, and then test it against the collaboration's published two-dimensional result. The central idea is simple: learn how much each simulated event should count, rather than first compressing events into a response matrix.

Start from familiar ground. We want a truth-level cross section, but observe selected reconstructed events. Smearing migrates events, inefficiency removes some, and backgrounds add others. Every unfolding method has to answer the inverse question at the bottom. OmniFold changes the representation of this problem, not the physical corrections that are required.

A response matrix works very well in modest dimensions. The structural problem is scaling: each new axis multiplies both truth and reconstructed bins. Yet the underlying simulation is not a matrix. It is a list of events, each with a truth record and a reconstructed record. OmniFold operates on that list before histogramming it.

The ML ingredient is density-ratio estimation. Under class-normalized training, the classifier odds p over one minus p estimate the local data-to-simulation ratio. This is not an interaction classifier and it is not being asked to label QE or resonance. It answers only: in this region of measured feature space, how much more or less data do I see than simulation?

Step 1 happens in detector space. I compare the selected data, after the background purity correction, with reconstructed signal simulation carrying the current push weights. The first classifier learns the data-to-MC correction. Because each simulated event is a paired truth and reco record, that correction attaches directly to the truth record. A truth signal event without a reconstructed partner is a miss; native OmniFold predicts its Step-1 weight from truth features using the reconstructed subsample.

Two classifiers per iteration: detector correction + truth regularization
Pullback alone would give weights only to the sampled truth events. Step 2 trains a second classifier between the original truth sample and that pulled-back weighted sample. It learns a smooth truth-space density ratio. Those are the new push weights. Push them forward through the detector simulation and repeat. This second classifier is important: it is the truth-level regularization and the mechanism that lets misses receive a consistent update.

After the final iteration, the product is not yet a histogram. It is a truth-level simulated event sample with learned push weights. I can choose the paper's binning, make one-dimensional projections, or change the bin edges after the fit. The careful version of “bin at the end” is that I can rebin and project truth features carried by the learned event representation; a genuinely new feature must be included or otherwise represented.

This is the concrete interface to the open product. My event loop writes four flat trees per playlist. Data provide reconstructed observations. Background MC supplies the subtraction. Signal reco contains paired reco and truth features. The truth denominator contains every true signal event and is authoritative: reconstructed signal is accepted only if its event key is in that denominator, and truth-only events are appended as native misses. The event loop must run separately for each of the twelve playlists because playlist-specific flux and calibration are selected from the first run; only then are the outputs merged.

The classifier is not the whole measurement. I first mask data to the signal phase space and subtract backgrounds through a non-negative per-bin purity weight. OmniFold returns push weights. The unfolded truth yield is the sum of push weight times the event's central truth weight. I then divide by the flux per POT, the data exposure in POT, the fixed tracker nucleon count, completeness and bin widths. Native misses make the truth ensemble acceptance-corrected; completeness is exactly one, so applying another absolute efficiency correction would double-correct the result.

No single validation proves an unfold. Closure tests whether injected changes are recovered. The 2D pull summary is a same-ensemble Gaussianity diagnostic, not coverage: the MC bootstrap also fluctuates the stored truth reference, so an independent-truth coverage test remains open. Iteration and classifier-family tests probe regularization dependence. The binned study connects the first update to D'Agostini IBU. The classifier AUC drop is also descriptive because the weights were not cross-fitted through the full OmniFold chain and no end-to-end null calibration was run.

This is the credibility test. I trained in the continuous two-dimensional muon-kinematic space and only afterward binned the weighted truth events into the paper's exact 14 by 16 grid. The integrated cross section is 1.1 percent high. The median bin differs by six-tenths of a percent, and 94.1 percent of reported bins are within ten percent. This is not input closure: it is a comparison to an independently published D'Agostini result on the same data.

AGREEMENT IS GOOD — BUT NOT FEATURELESS

Per-bin agreement and correlated shape tell different stories

Indicative paper-covariance distance: 3.66/bincorrelated low-pT shape modes; no shared-data cross-covariance
I do not want to oversell the word “reproduce.” Individual standardized differences have RMS 0.60, and the separately reconstructed uncertainty budget has essentially the same median size as the paper. But the indicative distance using the paper's full covariance is 3.66 per bin. The difference lives in correlated shape modes around the low-pT peak ridge rather than normalization. Because the two results share data and their cross-covariance is unavailable, this is not a calibrated compatibility test.

Keeping weighted events makes higher-dimensional measurements practical. I have added available energy, q3 and W, with each dimensional extension required to reproduce its predecessor under marginalization. At the level of current central values, the positive data-minus-generator difference concentrates at high available energy and high W; I am deliberately withholding significance while corrected N-dimensional uncertainties are regenerated. The longer-term path is to replace the scalar feature table with calorimeter-cluster point clouds.
